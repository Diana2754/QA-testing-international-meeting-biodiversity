## Objetivo

El objetivo de este plan de pruebas es validar el correcto funcionamiento
del sitio web del evento cultural internacional, asegurando que las
funcionalidades principales operen correctamente en diferentes
dispositivos y navegadores antes de su liberación.




## Alcance

Las pruebas incluirán:

- Navegación del sitio
- Carrusel de imágenes
- Información del evento
- Cronograma de actividades
- Sección de patrocinadores
- Sistema de donaciones con PayPal
- Formularios de contacto
- Responsive design
- Compatibilidad entre navegadores



## Fuera de Alcance

Las siguientes pruebas no serán cubiertas en esta fase:

- Pruebas de seguridad
- Pruebas de carga o performance avanzadas
- Automatización de pruebas




## Estrategia de Testing

Se realizarán pruebas manuales utilizando:

- Test cases
- Testing exploratorio
- Checklist de regresión

Se ejecutarán pruebas en diferentes resoluciones para validar el
comportamiento responsive del sitio.




## Ambiente de Testing

- Ambiente: Development
- Navegadores: Chrome, Firefox
- Dispositivos:
  - Desktop (1920px)
  - Tablet (768px)
  - Mobile (375px)



  ## Criterios de Salida

El testing se considerará finalizado cuando:

- Todos los test cases hayan sido ejecutados
- Los defectos críticos hayan sido corregidos
- Se haya realizado una ejecución completa de regresión



## Riesgos

- Integración incompleta con el sistema de donaciones PayPal
- Problemas de responsive en dispositivos móviles
- Cambios de diseño durante el proceso de testing



## Entregables

Durante el proceso de testing se generarán los siguientes documentos:

- Test cases
- Bug reports
- Regression checklist
- Regression execution report
- Test summary report
