QA Testing – International Cultural Event Website
Project Overview

This repository contains a manual QA testing project for a fictional International Cultural Event website.
The objective of this project is to simulate a real QA workflow including test planning, bug reporting, and regression testing.

The website includes common UI and functional components such as:

Event information

Image carousel

Event schedule (agenda)

Donation system using PayPal

Contact form

Responsive navigation

This project demonstrates practical QA skills such as test design, bug documentation, regression analysis, and structured reporting.

Testing Scope

The following areas of the application were tested:

Website navigation

Image carousel functionality

Event information content

Event schedule layout and readability

PayPal donation button integration

Contact form validations

UI consistency

Responsive behavior

Cross-browser compatibility

Basic accessibility

Initial regression execution

Final regression cycle

Example Defects Identified

Examples of defects discovered during testing:

Carousel navigation not working on mobile

PayPal donation button not redirecting

Schedule layout broken on tablet resolution

Contact form allowing invalid email submission

Each issue was documented following a bug reporting workflow similar to Jira-style defect tracking.
